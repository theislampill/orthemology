**Date:** 2026-07-20 · **Authority:** R7 owner authorization · **Status:** proposed-candidate · **Reopens nothing:** Decisions 0001–0020 stand; the school-neutral core, the verdict/notation registries, and the reason/revelation firewall are unchanged.

## Problem

The R6 integration note (`docs/integrations/daee-epistemics-orthemology-mapping.md`) treated the public `theislampill/daee-epistemics` repository as a donor of five generic engineering controls and rejected its deformation taxonomy wholesale. Read against the pinned repository, that is too shallow: daee describes itself (pinned commit `c86b3c6673147b8802fe222373a165a37d4d24a8`, `README.md`) as "a modular LLM skill and governed diagnostic framework for epistemic operations and noetic analysis… designed to examine the condition of the qalb (heart-mind) and the ʿaql (intellect) before replying to doubts", whose aim is "to restore sound cognition so that foundational knowledge, inference, testimony, signs, and revelation are encountered in their proper order", with a typed dispatch-gating **Diagnostic IR**, recursive state re-read, owner/TTP routing, a bounded release rubric, and an explicit **inference boundary**. That is a **domain-specific noetic application of the orthemma–ortheme–metaortheme architecture**, not merely engineering habits.

## Decision

1. A read-only, typed, two-way **application crosswalk** is added under `applications/daee-epistemics/` (`README.md`, `DAEE-ORTHEMOLOGY-CROSSWALK.yaml`, `NOETIC-ORTHING-APPLICATION.md`, `SOURCE-BOUNDARY.md`, `NOTATION-FIREWALL.yaml`, `EPISTEMOLOGICAL-AND-METAPHYSICAL-BOUNDARY.md`). Every daee-derived statement is tagged with one status layer: **direct-repository-description** / **orthemological-crosswalk** / **creed-internal-normative** / **empirical** (currently none). The application never validates its own premises.
2. Every mapped object records the daee source path + **pinned commit**, the daee name/symbol, a plain-language role, the Orthemology object, a mapping type (exact / partial / one-to-many / many-to-one / application-extension / no-mapping), analysis level, evidence status, non-claims, and any notation collision.
3. The crosswalk **distinguishes**, and never collapses: concrete engagement occurrence ≠ public discourse observation ≠ candidate noetic profiles ≠ inferred profile (`Ψᴵ`) ≠ actual interior condition ≠ governing metaorthemes ≠ case-bound metaorthemmata ≠ Diagnostic IR ≠ policy/TTP ≠ orthing episode ≠ runtime closure (`Ψᴺ`) ≠ rendered response ≠ actual uptake/restoration. `Ψᴵ` is never identified with ground truth or soul-state.
4. **Deformation taxonomy (`hawā`, `taqlīd`, `ʿāda`, `gharaḍ`, `shubhah`, …):** not imported into the school-neutral core as truth. Within the creed-internal application, its members are **candidate ortheme-level state-types under a declared noetic analysis**, requiring evidence, underdetermination, and evaluator symmetry; the rules that select and route them are **metaorthemic**; concrete evidence/threshold/scope/module bindings are **metaorthemmata**. Placements never assert motive, culpability, or soul-state.
5. **Notation firewall:** daee symbols (`μ` meta-noetic memetic vector, `κ` collapse radius, `Ω` ontological grammar, `m` deformation/memetic mode, `N` frame) collide with Orthemology's registry (`μ` metaortheme, `κ` identity key, `Ω` observation map, `m` orthemma). The crosswalk uses namespaced forms (`D.mu_mem`, `D.kappa_dep`, `D.Omega_ont`, `D.m_mode`, `D.N_frame`); the notation registries are **never merged**; daee's repository notation is not modified. `scripts/validate_daee_crosswalk.py` enforces this with tamper tests.
6. **Co-development limitation:** same owner, related model lineage, possible cross-influence; the application demonstrates **representability** and can **expose missing distinctions**, but is **not independent validation** and its apparent convergence is **not novelty evidence**.
7. Ten adversarial **noetic-application fixtures** (`tests/noetic-application-fixtures.yaml`, `examples/noetic-analysis-episode.json`, `scripts/validate_noetic_application.py`) assert underdetermination, sound-token/defective-binding separation, evaluator self-audit, false corroboration, correct-diagnosis-wrong-route, false closure, language-coupling-without-uptake, and a negative control against over-diagnosis — each asserting **no soul access**.

## Consequences

New `applications/daee-epistemics/` package, two validators, an example, a fixture file, CI steps, and the `EPISTEMOLOGICAL-AND-METAPHYSICAL-BOUNDARY.md` note. The R6 integration note is superseded by an additive dated banner (its body preserved). The manuscript and companions gain bounded cross-references identifying daee as a **domain-specific application and stress test, not evidence** and not part of the novelty proof.
