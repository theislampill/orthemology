# Primer — Arm C (coordinated vocabulary). Reading time budget: 3 minutes.

Five terms, defined once. They add no facts; they name distinctions you may use.

- **Orthemma** — a concrete situated case to be handled: *this* branch state, *this* batch, *this* position. It has an identity (which thing) and a version (which state of that thing). Evidence attaches to a version.
- **Ortheme** — a repeatable state-type a case may instantiate ("mislabeled", "seal-cracked", "clean"). One case can instantiate several at once (its *profile*); do not collapse them.
- **Metaortheme** — a repeatable governing standard/configuration consulted while handling cases: which evidence counts, when a version is stale, what may be called done. Distinct from the objective and from the step-by-step procedure.
- **Metaorthemma** — the case-bound *token* of a governing standard as applied to this case: which datum plane, which instrument, which calibration, which scope. A sound general standard can be applied through a defective binding.
- **Orthing** — one dated run of handling a case under governing standards. Its *result* (was the answer right?) and its *pathway* (was the process sound, and would it work on the neighboring case?) are separate verdicts.
