# Primer — Arm C (coined vocabulary)

Read this carefully. You will answer nine short workplace questions afterwards. Use the vocabulary below where it genuinely helps; do not force it.

## The vocabulary

1. **Orthemma.** A concrete situated case: this branch state, this returned batch, this scanned artifact, this chess position. An orthemma has an identity (which thing it is) and a version (which state of that thing). Evidence attaches to a specific version; when the version changes, earlier evidence does not carry over without a reason.

2. **Ortheme.** A repeatable state-type that a case can instantiate: mislabeled, seal-cracked, infected, stale-cache. One orthemma can instantiate several orthemes at once, and each instantiated ortheme can demand its own handling. A case's full set of instantiated orthemes is its profile.

3. **Orthing.** The governed act of resolving an orthemma into orthemes and handling it. An orthing episode can reach the right answer through a bad process, or the wrong answer through a good process: the result verdict and the process verdict are separate. A checker that passes today's artifact for a reason that would also pass a bad artifact has a defective process even when today's answer is right.

4. **Metaortheme.** A repeatable governing standard used during orthing: a freshness rule, a verification standard, an evidence threshold. When the same error keeps recurring across episodes, suspect the shared metaortheme rather than only patching each instance; revising the standard is a separate act from repairing an instance, and past decisions made under the old standard may need re-checking.

5. **Metaorthemma.** The case-bound configuration token that ties a metaortheme to one particular case: which datum plane, which instrument, which calibration, which source set, bound for this job. The general standard can be sound, the worker faithful, and the procedure well-formed while this bound record is wrong — a defect that lives neither in the standard nor in the execution.

6. **Completion discipline.** A claim that work is "done" quantifies over each obligation's own state — resolved, deferred, transferred, risk-accepted. Collapsing transferred or risk-accepted obligations into "resolved" is a false completion claim.

7. **Per-side evaluation.** One orthemma may be evaluated under two different analyses with opposed objectives (two chess engines on one position). "Winning for one side" and "losing for the other" is one fact under two indices, not a contradiction; shared standards of analysis govern how placements are made, never whose objective wins.

Answer plainly. Where a question has a simple one-step answer, give it; the vocabulary earns its place only where it prevents a real mistake.
