# Decision 0016 — Current review state, exact public-surface markers, and historical supersession

**Date:** 2026-07-20 · **Authority:** R5 owner authorization (final public-state/evidence-boundary tie-off) · **Status:** adopted · **Reopens nothing:** Decisions 0001–0015 all stand; this is an engineering/state decision, not a theory claim.

## Problem

After the fresh-session review of the mixed-provenance R4 candidate completed and PR #3 merged, every review-state surface the validators did **not** check remained stale: `STATUS.md` and all five primary document headers still said "candidate revision pending independent review", Decisions 0011–0014 still carried candidate-status headers, the changelog ended at R3, the fresh-review durable state file still said the merge was pending, and no in-repository record named the merge commit or its verification. A green validator run therefore coexisted with a false current-state claim — the exact stale-state/false-closure class this project exists to prevent (independent post-merge audit, findings B1/B6/B7/B8/M1/M3).

Root cause: the review-state surfaces were hand-maintained prose with no machine contract; Decision 0014's whole-state-reread rule was applied to every surface a validator checked, and to no surface it didn't.

## Decision

1. **The generated current state carries an authored `review_state` block** with stable, non-self-referential fields: `status` (closed vocabulary; currently `fresh-session-review-completed`), `scope`, `current_signoff` (path), `current_merge_verification` (path), `historical_status_index` (path), `empirical_validation`, `terminology_adoption`, and `header_wording` (the exact wording every current primary surface must carry). **No commit hash may appear in this block**: commit IDs and Actions run IDs live in the merge-verification record and Git history, never in a tracked equality contract (Decision 0014 amendment rationale).
2. **Exact public-surface markers.** `VERSION`, `README.md`, `STATUS.md`, every current primary document header, and the PDF status page (via `build_pdfs.py` `STATUS_LINES`) must carry the authored `header_wording` and must not carry any banned stale phrase ("candidate revision pending independent review", "REQUIRES INDEPENDENT REVIEW", "not independently signed off" as a current claim). The bounded current wording is: *current research draft; fresh-session repository review completed; not external human peer review; not empirically validated; no term adopted.* It never asserts peer review, publication readiness, empirical validation, or legal closure.
3. **Historical/current classification is machine-readable.** `docs/project-closure/HISTORICAL-STATUS-INDEX.yaml` classifies every project-closure artifact as `current`, `current-signoff`, `historical-snapshot`, `superseded`, or `interruption-provenance` (prefix rules with per-path overrides). Historical bodies remain intact; supersession is expressed by dated banners and the index, never by rewriting.
4. **Decision-header/status agreement.** A decision whose registry status is `adopted` may retain its historical "candidate revision requiring independent review" wording **only** together with a dated review-discharged notice naming the sign-off. Decisions 0011–0014 carry that notice.
5. **Owner burden discharge.** `R4-UNAVOIDABLE-OWNER-ACTIONS.md` item 7 (commission independent review; merging is an owner action) was **discharged** by the owner-commissioned fresh-session review and the protected merge of PR #3 (2026-07-20). The file remains as a historical snapshot with a dated banner; the current burden set is the six-item list in the generated state and `OPEN-DECISIONS.md`.
6. **Final merge-verification records.** `docs/project-closure/r5/FINAL-MERGED-VERIFICATION.md` + `FINAL-MERGED-STATE.json` carry the exact final commit/run/manifest/PDF identities. Until the R5 merge completes they are explicitly PROVISIONAL and claim nothing; they are finalized from live remote facts in a follow-up protected PR, so the record states the *previous* merge commit and its verification rather than its own future commit (no self-referential commit paradox).
7. **Validation.** `scripts/validate_review_state.py` (CI step) fails on: a stale/banned phrase on any current surface; a current surface missing the authored wording; a decision header contradicting the registry without a discharge notice; a missing/misclassified sign-off or verification path; an unclassified project-closure artifact; or a historical snapshot presented as current.

## Consequences

New: `docs/project-closure/HISTORICAL-STATUS-INDEX.yaml`, `docs/project-closure/r5/FINAL-MERGED-VERIFICATION.md`, `docs/project-closure/r5/FINAL-MERGED-STATE.json`, `scripts/validate_review_state.py`, CI step. Amended surfaces: `VERSION`, `README.md`, `STATUS.md`, five primary headers, Decisions 0011–0014 headers (additive notice), `build_pdfs.py` status lines, `R4-UNAVOIDABLE-OWNER-ACTIONS.md` (additive banner). The R4 fresh-review pre-merge state JSON is preserved unmodified as a historical snapshot.
