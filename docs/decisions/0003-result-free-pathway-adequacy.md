# Decision 0003 — Result-free pathway adequacy (O2)

**Date:** 2026-07-19 · **Decider:** project owner · **Implemented by:** Claude Fable 5 under owner direction · **Status:** implemented, machine-validated, and promoted (reconciliation R1).

**Question.** Should `PathwayAdequate` include result correctness (V1) and a factive truth-connection verdict — or be a result-free core, so that "right answer" and "properly run process" stay independent axes?

**Decision: (a) — result-free pathway core.** Three questions are kept distinct: (1) was the result correct (V1, judged against `O*(m; A(e))`); (2) was the *configured procedure* genuinely truth-conducive over its declared reference class (**V2b^proc** — non-factive; per-claim `RelSpec_q` with reference class, stratum, metric, threshold, perturbation family, protocol, reliability evidence, version/validity; the reference class is fixed before seeing the outcome; one correct result cannot establish it); (3) was *this* correct claim truth-linked rather than coincidental (**V2b^tok** — factive and claim-wise, `V2b^tok_q → V1_q`; excluded from the core precisely because of its factivity; reportable for stopped-clock/Gettier diagnoses). The pathway core is `{V2a, V2b^proc, V2c, V3, V3a, V3c, V3b, V3d, V4a, V5, V6}` — V1, V2b^tok, and V4b (advisory near-optimality) are excluded. Applicability is governance-derived (`ReqPath(e)`), never a discretionary list; "not tested" is never "not applicable"; every exclusion carries a reason. Statuses are four-valued (pass/fail/undetermined/not-applicable) with `PathwayAdequate` / `PathwayDefective` / `PathwayUndetermined`; a missing assessment is never a silent pass. The false "none entails another" sentence is replaced by a declared implication table whose only entailment is `V2b^tok_q → V1_q`. All four result × pathway cells are representable — including `AdequatePathError` (the justified rare miss of a reliable process; explicitly *not* a moral/legal/theological "blamelessness" predicate) — witnessed by deterministic fixtures F1–F5.

**Machine validation:** `tests/verdict-fixtures.json` + `scripts/validate_verdict_semantics.py` (29 checks: core exclusions, zero-burden V3c coupling, factivity, non-factivity witness, four-cell satisfiability, undetermined handling, N/A reasons).
