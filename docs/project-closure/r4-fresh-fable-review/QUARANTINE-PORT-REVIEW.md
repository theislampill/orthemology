## C5 review of the internal-reference validator (deferred here from Phase C)

Verified: local missing paths fail; relative Markdown links resolve from the
source document; code-block path mentions are validated as citations (they can
never become exemptions — exemptions come only from the reviewed YAML);
external exemptions name their owning repository; retired paths must name an
existing successor and a reason; stale exemptions (cited nowhere) fail;
exemption matching is exact set membership, never prefix-broad.

Two gaps found against the required properties, both hardened in the ported
copy:

1. **External shadowing**: nothing prevented an `external_paths` entry from
   naming a path that exists locally (which would mask that local target's
   future disappearance). Now: an external exemption that resolves locally
   fails.
2. **Repository escape**: a relative link normalizing above the repository
   root was silently skipped. Now: any cited path that escapes the root is a
   failure (it cannot be a valid internal reference).

Accepted limitation (noted): the stale-exemption liveness count uses a
substring-count heuristic that subtracts the registry's own listing lines;
adequate for the small fixed registry, and a false positive fails toward
review, not toward silence.
